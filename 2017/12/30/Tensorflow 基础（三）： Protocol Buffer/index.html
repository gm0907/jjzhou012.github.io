序列化

当要将结构化的数据持久化或者进行网络传输时，首先要将它们序列化。
序列化：将结构化的数据变成数据流的格式。
处理结构化数据： 将结构化的数据序列化，并从序列化之后的数据流中还原出原来的结构化数据。

除 Procotol Buffer 之外，XML 和 JSON 是两种常用的结构化数据处理工具。

Procotol Buffer 格式的数据和以上两者有较大区别。

首先，Procotol Buffer 序列化之后得到的数据不是可读的字符串，而是二进制流。
其次，XML 或 JSON 格式的数据信息都包含在序列化之后的数据中，不需要任何其他的信息就能还原序列化之后的数据。但使用 Procotol Buffer 时需要先定义数据格式 （schema）。还原一个序列化之后的数据需要使用这个定义好的数据格式。这样定义的差别，Procotol Buffer 序列化出来的数据比 XML 格式的数据小3到10倍，解析时间快20到100倍。

Procotol Buffer 定义数据格式的文件一般保存在 .proto 文件中。

每一个 message 代表了一类结构化的数据。message 里面定义了每一个属性的类型和名字。
Procotol Buffer 里属性的类型可以是布尔型、整数型、实数型、字符型等基本类型，也可以是另一个 message。
在 message 中，Procotol Buffer 也定义了属性的性质：

  • required：必须的， 所有该 message 的实例必须要有这个属性；
  • optional： 可选的， 这个属性的取值可以为空；
  • repeated： 可重复的， 这个属性的取值可以为列表；
